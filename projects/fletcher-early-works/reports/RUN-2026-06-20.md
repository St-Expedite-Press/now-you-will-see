# Run Report — Station End-to-End (Phase 6)

**Date:** 2026-06-20
**Scope:** Drive the reference tenant `fletcher-early-works` through the publishing
station end to end — every screen's **gated specialist agent** executing **real
tools** through the agent runtime, with gating and gate/hand-off enforced against
the live `PROMOTION.yaml` files.

**Driver:** `tools/station_e2e.py` (`--full` builds the proof). Deterministic — a
scripted adapter stands in for the LLM so the *decisions* are fixed; the *execution*
is the real `texgraph` CLI against the live project. Regression-guarded by
`backend/tests/test_station_e2e.py` (fast; faked execution, real gate files).

### Gating enforced (browser → API → runtime → CLI)

> interior agent asked to `texgraph ingest rename` (a **sources** tool)
> → `allowed=False` · `REFUSED: 'texgraph ingest rename' is not permitted for the interior stage.`

The tool was **not executed**. A gated agent can only call the tools its
`framework/agents/<stage>/agent.yaml` allow-list grants.

### Gate / hand-off chain (read from the live `PROMOTION.yaml`)

```
workspace      gate PASSED   (no gate)
sources        gate PASSED   (sources/PROMOTION.yaml      = approved)
transcription  gate PASSED   (transcription/PROMOTION.yaml = approved)
manuscript     gate PASSED   (editorial layer, no gate)
interior       gate pending  (interior not yet signed off)
```

The chain advances Project → Acquire → Structure → Proof and **correctly stops** at
the interior gate: the volume's proof has not been approved, so the runtime would
refuse to hand off downstream. This is the true editorial state of the volume
(`manuscript/PROMOTION.yaml: pending`, awaiting the human proof pass).

## What this proves

- The three portions connect: **frontend → `/api/agent/stage` → `runtime_service`
  → `AgentRuntime` → `ToolCatalog` → `texgraph` CLI → real artifacts.**
- **Gating is real and enforced in code**, not by prompt convention — an
  out-of-scope tool is refused before execution.
- **Gates are real** — hand-off is governed by the actual `PROMOTION.yaml` files.
- The reference edition is intact: coverage is **1:1** and the interior proof
  **compiles** when the typeset agent calls `proof-build`.
